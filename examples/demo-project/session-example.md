# 示例会话

下面是一个推荐的首轮用户输入示例。

## 用户输入

请作为投标经理读取当前工作区材料，先完成项目启动咨询，再判断招标文件是否多包，生成解析页、评分点-章节-证据映射表和目录占位；如项目较复杂，可自行启用内部 sub-agent。

## 预期 manager 行为

1. 明确当前阶段：项目启动
2. 先问最少关键问题：
   - 投标主体是谁
   - 本次角色是什么
   - 必须纳入哪些厂商/证据
   - 哪些不能写太满
3. 检查工作区是否存在 `bid-vault/`
4. 进入招标解析阶段，判断是否多包
5. 若多包，先要求确认目标包件
6. 形成解析页与映射页
7. 生成目录占位
8. 请求用户确认目录

## 预期质量标准

- 不在目录确认前写完整正文
- 不把原厂能力写成投标人自有能力
- 不在无证据情况下输出正式资格结论
- 不暴露内部多 agent 编排细节给用户，除非用户主动询问
